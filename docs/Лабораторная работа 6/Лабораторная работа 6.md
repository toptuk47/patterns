# Отчет Лабораторная работа 6

---

## Задача

Разработать UML-диаграммы (диаграмму классов и диаграмму последовательности), и, с помощью паттерна **"Observer"** решить следующую задачу.

Деканат отлеживает текущую успеваемость в группах факультета по одной из дисциплин. Преподаватели раз в неделю создают текущую успеваемость и размещают ее в базе данных. Если преподаватель вовремя не создал текущую успеваемость - деканат оповещает об этом кафедру.

## Результат

**Диаграмма классов:**

```mermaid
classDiagram
    class Subject {
        <<interface>>
        +Attach(Observer)
        +Detach(Observer)
        +Notify()
    }

    class Observer {
        <<interface>>
        +Update(Data)
    }

    class PerformanceTracker {
        -teachers: Map~string, boolean~
        -deadlinePassed: boolean
        +Attach(Observer)
        +Detach(Observer)
        +Notify()
        +SubmitReport(teacherId)
        +CheckDeadline()
    }

    class DeanOffice {
        +Update(Data)
    }

    Subject <|-- PerformanceTracker
    Observer <|-- DeanOffice
    PerformanceTracker --> Observer : observers
```

**Диаграмма последовательности:**

```mermaid
sequenceDiagram
    participant Teacher1
    participant Teacher2
    participant PerformanceTracker
    participant DeanOffice

    Teacher1->>PerformanceTracker: SubmitReport("teacher1")
    Note over PerformanceTracker: teacher1 = true
    Note over Teacher2: не сдал отчёт
    Note over PerformanceTracker: наступает дедлайн
    PerformanceTracker->>PerformanceTracker: CheckDeadline()
    PerformanceTracker->>PerformanceTracker: определить несдавших (teacher2)
    PerformanceTracker->>DeanOffice: Update(["teacher2"])
    DeanOffice->>DeanOffice: оповестить кафедру о teacher2
```

## Контрольные вопросы

1. С помощью каких еще паттернов проектирования можно решить поставленную задачу?

Эту задачу можно также решить с помощью паттерна **Цепочка обязанностей**: при проверке того, сдали ли преподаватели текущую успеваемость, можно гибко настроить последовательность обработки событий, вместо того, чтобы создавать централизованный список подписчиков.
