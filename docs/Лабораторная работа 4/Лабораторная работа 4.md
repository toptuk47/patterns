## Задание

Разработать UML-диаграмму (диаграмму классов и диаграмму последовательности), и, с помощью паттерна **"proxy"** решить следующую задачу.

Создать простейшую модель фрагмента графического редактора, позволяющую нарисовать на экране монитора бокс, имеющий размеры реального изображения, хранящегося на диске под именем `TestImage`. Используя паттерн **"proxy"** обеспечить свободное перемещение бокса с помощью **"мыши"** по экрану. При двойном нажатии на правую клавишу **"мыши"** обеспечить загрузку реального изображения в нарисованный бокс.

## Диаграмма классов

```mermaid
classDiagram
    class Graphic {
        <<interface>>
        +draw(ctx: CanvasRenderingContext2D, x: number, y: number)
        +getWidth() number
        +getHeight() number
    }

    class RealImage {
        -image: HTMLImageElement
        -loaded: boolean
        +RealImage(filename: string, width: number, height: number)
        +draw(ctx, x, y)
        +getWidth() number
        +getHeight() number
    }

    class ImageProxy {
        -realImage: RealImage | null
        -filename: string
        -width: number
        -height: number
        +ImageProxy(filename: string, width: number, height: number)
        +draw(ctx, x, y)
        +loadImage()
        +getWidth() number
        +getHeight() number
    }

    class EditorClient {
        -proxy: ImageProxy
        -x: number
        -y: number
        +startDrag(mx, my)
        +moveDrag(mx, my)
        +endDrag()
        +handleRightDoubleClick()
        +render()
    }

    Graphic <|.. RealImage
    Graphic <|.. ImageProxy
    ImageProxy o-- RealImage : realImage (nullable)
    EditorClient --> Graphic : uses
```

## Диаграмма последовательности

```mermaid
sequenceDiagram
    participant User
    participant EditorClient
    participant ImageProxy
    participant RealImage

    User->>EditorClient: начать перетаскивание (mousedown)
    EditorClient->>EditorClient: запомнить смещение
    loop перемещение мыши (mousemove)
        User->>EditorClient: двигать мышь
        EditorClient->>EditorClient: обновить координаты (x,y)
        EditorClient->>ImageProxy: draw(ctx, x, y)
        alt реальное изображение отсутствует
            ImageProxy->>ImageProxy: рисовать пунктирный бокс
        end
    end
    User->>EditorClient: отпустить кнопку мыши (mouseup)
    User->>EditorClient: двойной правый клик по боксу
    EditorClient->>ImageProxy: loadImage()
    ImageProxy->>RealImage: new RealImage("TestImage", w, h)
    RealImage->>RealImage: загрузить TestImage с диска
    EditorClient->>ImageProxy: draw(ctx, x, y) (следующий кадр)
    ImageProxy->>RealImage: draw(ctx, x, y)
    RealImage->>RealImage: отрисовать загруженное изображение
```

## Контрольные вопросы

1. Чем похожи и чем отличаются паттерны **Proxy**, **Adapter** и **Decorator**?

Все три паттерна - **структурные**, оборачивающие объект.

- **Proxy** - оборачивает объект, контролируя доступ к нему.
- **Adapter** - приводит чужой интерфейс к ожидаемому клиентом.
- **Decorator** - добавляет новые обязанности динамически, не меняя интерфейс.
