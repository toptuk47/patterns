# Отчет Лабораторная работа 2

---

## Цель работы

Применение паттерна проектирования Builder (строитель)

## Задание

Разработать UML-диаграммы (диаграмму классов и диаграмму последовательности), и, с помощью паттерна "**Строитель**" решить следующую задачу.

**Задача.** Обеспечить контроль загрузки и готовности к отправлению автобусов и такси. Водитель такси и автобуса имеют права разной категории. Без водителя машина не поедет. Два водителя в одну машину сесть не могут. Без пассажиров машины не поедут. Есть лимит загрузки машин. Для автобуса - 30 человек. Для такси - 4 человека.

Есть разница между пассажирами автобуса и такси.

Для автобуса три категории пассажиров:

- Взрослый
- Льготный
- Ребенок

Для всех разная стоимость билета.

Для такси:

- Взрослый
- Ребенок

Необходимо детское кресло.

## Результат

Диаграмма классов:

```mermaid
---
title: Подготовка автобусов и такси
---
classDiagram
    class Director {
        +constructBus(Driver, Passenger[])
        +constructTaxi(Driver, Passenger[])
    }
    Director --> VehicleBuilder: использует
    
    class VehicleBuilder {
        <<interface>>
        +setDriver(Driver)
        +addPassenger(Passenger)
        +isReadyToDepart() bool
        +getVehicle() Vehicle
    }
    
    class BusBuilder {
        -Bus bus
        +setDriver(Driver)
        +addPassenger(Passenger)
        +isReadyToDepart() bool
        +getVehicle() Vehicle
    }
    
    class TaxiBuilder {
        -Taxi taxi
        +setDriver(Driver)
        +addPassenger(Passenger)
        +isReadyToDepart() bool
        +getVehicle() Vehicle
    }
    
    class Vehicle {
        <<interface>>
        +addDriver(Driver)
        +addPassenger(Passenger)
        +getCurrentLoad() int
        +getMaxCapacity() int
        +isReadyToDepart() bool
    }
    
    class Bus {
        -List~Passenger~ passengers
        -Driver driver
        -int maxCapacity = 30
        +addDriver(Driver)
        +addPassenger(Passenger)
        +getCurrentLoad() int
        +getMaxCapacity() int
        +isReadyToDepart() bool
    }
    
    class Taxi {
        -List~Passenger~ passengers
        -Driver driver
        -int maxCapacity = 4
        -bool hasChildSeat
        +addDriver(Driver)
        +addPassenger(Passenger)
        +getCurrentLoad() int
        +getMaxCapacity() int
        +isReadyToDepart() bool
        +isChildSeatNeeded() bool
    }
    
    class Passenger {
        -PassengerCategory category
        +getCategory() PassengerCategory
    }
    
    class PassengerCategory {
        <<enumeration>>
        ADULT
        PREFERENTIAL
        CHILD
    }
    
    class Driver {
        -DriverLicense license
        +getLicense() DriverLicense
    }
    
    class DriverLicense {
        <<enumeration>>
        BUS
        TAXI
    }
    
    Vehicle <|-- Bus
    Vehicle <|-- Taxi
    VehicleBuilder <|-- BusBuilder
    VehicleBuilder <|-- TaxiBuilder
    BusBuilder --> Bus: создает
    TaxiBuilder --> Taxi: создает
    Bus --> Passenger: содержит
    Bus --> Driver: содержит
    Taxi --> Passenger: содержит
    Taxi --> Driver: содержит
    Passenger --> PassengerCategory
    Driver --> DriverLicense
```

Диаграмма последовательности:

```mermaid
sequenceDiagram
    title Сценарий подготовки автобуса к отправлению
    
    actor Dispatcher as Диспетчер (Client)
    participant Director as Director
    participant Builder as BusBuilder
    participant Vehicle as Bus
    
    %% 1. Создание строителя
    activate Dispatcher
    Dispatcher ->> Builder: new BusBuilder()
    activate Builder
    note over Builder: Создает пустой объект Bus
    deactivate Builder
    
    %% 2. Передача строителя директору
    Dispatcher ->> Director: new Director(busBuilder)
    activate Director
    note over Director: Директор получает управление над строителем
    deactivate Director
    
    %% 3. Команда на постройку автобуса
    Dispatcher ->> +Director: constructBus(driver, passengers)
    
    %% 4. Установка водителя (проверка прав)
    Director ->> +Builder: setDriver(driver)
    activate Builder
    note over Builder: Проверяет, что водитель имеет права категории BUS
    
    Builder ->> Builder: Проверка DriverLicense
    
    Builder ->> +Vehicle: addDriver(driver)
    activate Vehicle
    note over Vehicle: Сохраняет водителя, если место свободно
    Vehicle -->> Builder: Водитель установлен
    deactivate Vehicle
    
    Builder -->> Director: 
    deactivate Builder
    
    %% 5. Добавление пассажиров в цикле
    loop для каждого пассажира из списка
        Director ->> +Builder: addPassenger(passenger)
        activate Builder
        
        Builder ->> +Vehicle: addPassenger(passenger)
        activate Vehicle
        
        note over Vehicle: Проверка: не превышен ли лимит?<br/>Для автобуса макс. 30
        
        Vehicle -->> Builder: Пассажир добавлен
        deactivate Vehicle
        
        Builder -->> Director: 
        deactivate Builder
    end
    
    %% 6. Проверка готовности к отправлению
    Director ->> +Builder: isReadyToDepart()
    activate Builder
    
    Builder ->> +Vehicle: isReadyToDepart()
    activate Vehicle
    note over Vehicle: Проверка:<br/>- Есть водитель<br/>- Есть пассажиры<br/>- Не превышен лимит
    Vehicle -->> Builder: true/false
    deactivate Vehicle
    
    Builder -->> Director: true
    deactivate Builder
    
    Director -->> Dispatcher: 
    deactivate Director
    
    %% 7. Получение готового транспортного средства
    Dispatcher ->> +Builder: getVehicle()
    activate Builder
    
    Builder -->> Dispatcher: bus (готов к отправке)
    deactivate Builder
    
    note over Dispatcher, Vehicle: Автобус полностью готов<br/>Водитель с правами BUS установлен<br/>Все 30 пассажиров добавлены
    deactivate Dispatcher
```
